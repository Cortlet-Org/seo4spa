## Contributing

Contributions are welcome!

If you want to add new features, fix bugs, or improve documentation:

1. Fork the repository
2. Create a new branch
3. Make your changes
4. Submit a Pull Request

Make sure your code passes all tests before submitting.

We will review your changes and merge them if everything looks good.

If your contribution is accepted, you will receive a **PR ID** (a 4-digit reference number).  
The merge commit message will include your PR ID for tracking and credit.

---

### Contribution Guidelines

Please follow these guidelines to keep the project clean and easy to maintain:

- **Write clear, concise commit messages.**
- **Follow the existing code style** used throughout the project.
- **Include unit tests** for any new features or bug fixes.
- **Update documentation** if your changes modify behavior or add features.
- **Be respectful and considerate** in all interactions.
- **Feel free to ask questions** if you need help — we’re happy to assist!
